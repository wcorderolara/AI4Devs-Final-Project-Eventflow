## Updated Product Owner Decisions to Apply

Use `/docs/8.1-Product-Owner-Decisions-Use-Cases-Addendum.md` as the main source for these decisions:

|  # | Decision Area                       | Decision to Apply                                                                                                                                            |
| -: | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|  1 | Review rating scale                 | Rating scale is **1 to 5**, where 5 is the highest rating and 1 is the lowest.                                                                               |
|  2 | Vendor portfolio images             | Vendor may upload up to **10 images per event/work shown** in the portfolio section.                                                                         |
|  3 | Vendor category changes             | Vendor may change/edit categories up to **5 times**. Substantive category changes should be reviewed or validated by admin if they affect public visibility. |
|  4 | Quote validity                      | If no validity date is specified, a Quote is valid for **15 calendar days**.                                                                                 |
|  5 | BookingIntent cancellation          | Confirmed BookingIntent can be cancelled in MVP because no payments are processed in platform. Penalties depend on external agreement with vendor.           |
|  6 | Automatic event completion          | Event is automatically marked as `completed` **2 days after event date**.                                                                                    |
|  7 | Currency change policy              | Event currency cannot be changed after event creation. During creation, user must choose local currency or USD.                                              |
|  8 | Captcha / anti-bot                  | Captcha and anti-bot protection must be included for registration and login.                                                                                 |
|  9 | AI timeout                          | AI timeout is **1 minute** before controlled error/fallback/MockAIProvider behavior.                                                                         |
| 10 | Admin dashboard metrics             | Admin panel should include operational/demo metrics, not commercial revenue metrics.                                                                         |
| 11 | Deleted reviews                     | Deleted reviews must be audited. Recommended behavior: soft delete / hidden status.                                                                          |
| 12 | QuoteRequest active limit           | Recommended limit: maximum **5 active QuoteRequests per service category per event**.                                                                        |
| 13 | Quote rejected/expired notification | Provider must be notified in platform; email notification applies when email functionality exists.                                                           |
| 14 | Vendor response to reviews          | Fully deferred to future scope. Not part of MVP.                                                                                                             |
| 15 | AnthropicProvider                   | MVP only requires provider interface prepared. OpenAIProvider functional, MockAIProvider functional, AnthropicProvider future/stub optional.                 |
| 16 | Admin event visibility              | Admin may list events and see general status for demo/support/governance, but cannot edit organizer-owned events in MVP.                                     |
| 17 | Admin EventType management          | Admin may manage EventTypes in a controlled way; no hard delete if EventType has associated events.                                                          |
| 18 | Category hierarchy                  | Service categories may support simple hierarchy up to **2 levels**. No deep hierarchies in MVP.                                                              |
| 19 | Attachment delete policy            | Use soft delete for attachments metadata; physical deletion may happen later via maintenance/technical process.                                              |

## Document-Specific Update Instructions

### 1. Update `/docs/4-Business-Rules-Document.md`

Review and update business rules related to:

#### Reviews

Add or update rules for:

* Rating scale from 1 to 5.
* Rating 5 is the highest.
* Rating 1 is the lowest.
* Admin deletion/removal must be audited.
* Recommended behavior: soft delete or hidden status.
* Vendor response to reviews is future scope.

#### Vendor portfolio

Add or update rules for:

* Vendor portfolio can include up to 10 images per event/work shown.
* Attachment delete should use soft delete.
* Physical deletion from storage can be handled later by maintenance.

#### Vendor categories

Add or update rules for:

* Vendor may edit/change categories up to 5 times.
* Category changes that affect public visibility may require admin review.
* Admin category hierarchy supports max 2 levels.

#### Quotes

Add or update rules for:

* Default Quote validity is 15 calendar days.
* Maximum 5 active QuoteRequests per service category per event, if applying the recommended rule.
* Quote expiration/rejection should update status and notify vendor in platform.
* Email notification is future/simulated depending on notification capability.

#### BookingIntent

Add or update rules for:

* Confirmed BookingIntent can be cancelled in MVP.
* Cancellation has no platform financial penalty.
* Any penalty depends on external agreement between organizer and vendor.
* BookingIntent does not represent payment or legal contract.

#### Events

Add or update rules for:

* Event auto-completes 2 days after event date.
* Admin may list events and view general status.
* Admin cannot edit organizer-owned events in MVP.
* Event currency is immutable after creation.

#### Currency

Add or update rules for:

* Event currency must be selected during event creation.
* User must choose local currency or USD.
* Currency cannot be changed after event creation.
* Currency conversion is out of scope.

#### Authentication

Add or update rules for:

* Captcha / anti-bot required for registration.
* Captcha / anti-bot required for login.

#### AI

Add or update rules for:

* AI timeout is 1 minute.
* After timeout, system must show controlled error or use fallback/MockAIProvider in demo/testing mode.
* AnthropicProvider is not required as functional MVP provider.
* AIProvider interface should support future providers.
* OpenAIProvider is functional MVP provider.
* MockAIProvider supports testing/demo.

#### Admin

Add or update rules for:

* Admin dashboard metrics focus on activity, governance, IA usage, quotes, demo readiness.
* Admin can manage EventTypes in controlled way.
* Admin cannot hard delete EventType if associated with events.
* Admin can manage simple category hierarchy up to 2 levels.
